# オセロ棋譜検討アプリ

ブラウザだけで動く、オセロの対局・CPU対戦・棋譜保存・棋譜再生・AI評価・悪手一覧・テンプレ解説アプリです。

## 実行方法

このフォルダを開き、`index.html` をブラウザで開いてください。

ローカルサーバーで開く場合は、このフォルダで次のように配信できます。

```sh
python3 -m http.server 8000
```

その後、ブラウザで `http://127.0.0.1:8000/index.html` を開きます。

## 操作方法

- 黒が先手です。
- 「2人対戦」と「CPU対戦」を切り替えられます。CPU対戦では開始時に黒か白を選べます。
- CPU難易度は easy / normal / hard から選べます。
- 盤面上の薄い丸が合法手です。
- マスをクリックすると石を置き、挟んだ石が反転します。
- 合法手がない場合は自動でパスし、パスも棋譜に記録されます。
- 「リセット」で最初から対局できます。

## 通常対局モード

8x8の通常オセロを対局できます。2人対戦とCPU対戦に対応し、現在の手番、黒白の石数、勝敗を表示します。
各手は `moveHistory` に保存され、盤面の前後、実際の手、反転した石、合法手、石数を記録します。

## CPU対戦モード

CPU対戦では開始時に黒か白を選べます。黒を選ぶと人間が先手、白を選ぶとCPUが黒で先に着手します。
人間が打つと、CPUの手番で約0.5秒後に自動で着手します。
CPUの手も人間の手と同じ形式で `moveHistory` に保存されます。検討モードではCPU手を飛ばし、人間の手だけを見返せます。

- easy: 合法手からランダムに選びます。
- normal: 角、Xマス、Cマス、相手への角献上、合法手数、返す石数を見て選びます。
- hard: 既存の negamax + alpha-beta の探索AIで推奨手を選びます。

CPU難易度が hard の時だけ、AI探索深さを選べます。

- 4: 標準
- 6: 強め。少し重くなることがあります。
- 8: かなり強め。局面によって重くなることがあります。

## 検討モード

「検討する」を押すと、棋譜を1手ずつ再生できます。

- 最初へ
- 前の手
- 次の手
- 最後へ
- 対局画面に戻る

対局中はAI評価、候補手ランキング、悪手一覧、解説を表示せず、盤面と対局操作に集中できます。
検討中は盤面クリックでは石を置けません。実際に打った手は青、AI推奨手は黄でハイライトされます。
CPU対戦ではCPUが打った手をナビゲーションから飛ばし、人間の手だけを表示します。
2人対戦では黒白どちらの手もこれまで通り評価できます。候補手ランキングは上位3手まで表示します。

## JSON読み込み・書き出し

対局操作から、現在の棋譜をJSONで書き出せます。ファイル名は `reversi-record-YYYYMMDD-HHMMSS.json` 形式です。
書き出しデータには `version`、`app`、`title`、`createdAt`、`moves` に加えて、壊れない範囲で手順情報や解析情報も含めます。

`.json` ファイルを読み込むと、`moves` を使って盤面と棋譜を復元し、検討モードに入ります。
読み込み時は拡張子、1MB以下のサイズ、JSON形式、`moves` 配列、手の形式、実際の合法手順を検証します。

## AI評価の意味

最善手、scoreLoss、accuracy、reasonTags はすべてローカルJavaScriptのオセロAIで計算します。
GPTには最善手判断をさせない設計です。

AIは negamax + alpha-beta pruning で候補手を探索します。CPU難易度 hard では探索深さを4、6、8から選べます。
空きマスが14以下になると終盤モードとして最後まで読みます。

## scoreLoss / accuracy / 判定

- `scoreLoss`: AI推奨手の評価値と、実際に打った手の評価値の差です。
- `accuracy`: `100 - scoreLoss` を0〜100に丸めた目安です。
- 判定分類:
  - 0〜5: 最善級
  - 6〜15: ほぼ問題なし
  - 16〜35: 疑問手
  - 36〜70: 悪手
  - 71以上: 大悪手
