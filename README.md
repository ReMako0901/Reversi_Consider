# オセロ棋譜検討アプリ

ブラウザだけで動く、オセロの対局・CPU対戦・棋譜検討アプリです。  
対局後に1手ずつ盤面を振り返り、ローカルAIによる評価、候補手ランキング、悪手一覧、解説文、対局全体の総評を確認できます。

GitHub Pagesで公開する想定の静的Webアプリです。サーバーやDBは使わず、HTML / CSS / JavaScriptだけで動作します。

## 主な機能

- 8x8の通常オセロ対局
- 2人対戦
- CPU対戦
- CPU対戦開始時の黒/白選択
- CPU難易度 easy / normal / hard
- hard時の探索深さ 4 / 6 / 8
- 合法手表示、自動パス、終局判定
- 棋譜保存と1手ずつの再生
- AI評価、accuracy、scoreLoss、判定表示
- 候補手ランキング上位3手
- 悪手一覧
- 1手ごとのルールベース解説
- 対局全体の総評
- 「次の分岐へ」によるAI推奨手との差分ジャンプ
- JSON形式の棋譜読み込み・書き出し
- PC/スマホ対応を意識したレスポンシブUI

## ファイル構成

- `index.html`: 画面構造
- `style.css`: レイアウト、デザイン、レスポンシブUI
- `main.js`: オセロのルール、CPU、棋譜検討、AI評価、解説生成、JSON入出力

## 実行方法

`index.html` をブラウザで開くと動作します。

ローカルサーバーで確認する場合は、このフォルダで次を実行してください。

```sh
python3 -m http.server 8000
```

その後、ブラウザで以下を開きます。

```text
http://127.0.0.1:8000/index.html
```

## 対局モード

対局中は盤面を中心に表示し、AI評価・候補手・悪手一覧・解説などの分析情報は表示しません。

基本操作:

- 黒が先手です。
- 盤面上の薄い丸が合法手です。
- マスをクリックすると着手し、挟んだ石が反転します。
- 合法手がない場合は自動でパスします。
- パスも棋譜に記録されます。
- 「リセット」で最初から対局できます。
- 対局後、「検討する」で棋譜検討に入れます。

## CPU対戦

CPU対戦では、開始時に盤面上のオーバーレイで自分の色を選びます。

- 黒で先手
- 白で後手

CPU難易度:

- `easy`: 合法手からランダムに選びます。
- `normal`: 角、Xマス、Cマス、合法手数、返す石数などを見る簡易評価です。
- `hard`: negamax + alpha-beta pruning による探索AIです。

`hard` の時だけAI探索深さを選べます。

- `4`: 標準
- `6`: 強め
- `8`: かなり強め

探索AIはブラウザ内で同期的に動くため、深い探索や長い棋譜では重くなることがあります。終盤完全読みは重くなりすぎないよう、空きマスが少ない局面に限定しています。

CPU対戦後の検討では、CPUの手は評価対象から外し、人間の手だけをテンポよく見返せます。

## 検討モード

「検討する」を押すと、棋譜を1手ずつ再生できます。検討モードでは盤面クリックで石は置けません。

操作:

- 最初へ
- 前の手
- 次の手
- 最後へ
- 次の分岐へ
- 対局画面に戻る

キーボード操作:

- 左キー: 前の手
- 右キー: 次の手
- Home: 最初へ
- End: 最後へ

盤面上では、実際に打った手とAI推奨手をハイライトします。  
「次の分岐へ」は、実際の手とAI推奨手がズレた次の手へジャンプします。

検討モードに入ると全体解析が走ります。解析中は画面全体にオーバーレイを表示し、操作を受け付けないようにしています。

## AI評価

各手について、ブラウザ内のローカルAIで評価します。外部AI APIには依存していません。

表示する主な情報:

- 実際の手
- AI推奨手
- accuracy
- scoreLoss
- 判定
- reasonTags
- 候補手ランキング

判定の目安:

- `0〜5`: 最善級
- `6〜15`: ほぼ問題なし
- `16〜35`: 疑問手
- `36〜70`: 悪手
- `71以上`: 大悪手

評価で見ている主な要素:

- 角
- Xマス
- Cマス
- 辺
- 合法手数
- 相手の合法手数
- 返しすぎ
- 安定石
- 終盤の石数
- 終局時の石差

## 候補手ランキングと悪手一覧

候補手ランキングでは、その局面の候補手を上位3手まで表示します。

悪手一覧では、評価損の大きかった手をまとめて表示します。  
悪手一覧の項目をクリックすると、その手へジャンプできます。

## 解説文

各手には、ローカルのルールベースで生成した解説文を表示します。  
単なるテンプレートではなく、盤面情報から主な理由を選び、2〜4文程度で説明します。

解説で扱う主な観点:

- なぜ良い手/悪い手なのか
- AI推奨手との違い
- 角、Xマス、Cマスの危険
- 自分と相手の合法手数の変化
- 返しすぎ
- 終盤の石数損
- 次回意識したい学習ポイント

例:

```text
ここは少しもったいない選択でした。実際の手はB3、AIの推奨はC4です。
この手で相手の打てる場所が5手から6手に増え、主導権を相手に渡しやすい形になりました。
AI推奨のC4なら相手の選択肢を2手に絞れていました。
序盤・中盤は石数より選択肢の多さが力になります。
```

## 総評

検討モードでは、全体解析後に1局全体の総評も表示します。

総評に含まれる内容:

- 対局結果
- 全体の流れ
- 勝敗を分けた手
- 勝因
- 次回の改善ポイント

総評では、単に最大scoreLossの手を見るだけではなく、以下の要素も見ます。

- 角を渡した手
- Xマス / Cマスに入った手
- 相手の合法手を大きく増やした手
- 自分の合法手を大きく減らした手
- 終盤で大きく損した手
- 連続した悪手
- 最終石数

CPU対戦では、人間側の振り返りを優先します。  
2人対戦では、黒白を中立的に扱います。

## JSON読み込み・書き出し

対局設定の「棋譜ファイル（JSON）」から、棋譜を読み込み・書き出しできます。

書き出しファイル名は次の形式です。

```text
reversi-record-YYYYMMDD-HHMMSS.json
```

JSONには主に以下を含みます。

```json
{
  "version": 1,
  "app": "Reversi_Consider",
  "title": "棋譜名",
  "createdAt": "ISO形式の日時",
  "moves": ["f5", "d6", "c3", "d3"],
  "gameMode": "cpu",
  "humanPlayer": "black",
  "cpuPlayer": "white"
}
```

読み込み時の検証:

- 拡張子が `.json`
- ファイルサイズが1MB以下
- JSONとしてパースできる
- 一番外側がオブジェクト
- `moves` が存在する
- `moves` が配列
- 手数が1〜60
- 各手が文字列
- 各手が `[a-h][1-8]` 形式
- 既存の合法手判定で、その順番で実際に打てる

不正なJSONの場合、検討処理には進まず、原因を表示します。

CPU対戦のJSONを読み込んだ場合はCPU対戦情報を復元し、検討時にCPUの手をスキップします。

## セキュリティ方針

- JSON由来の文字列は信用しません。
- `title` は最大50文字程度に制限します。
- `moves` は形式と合法手順を検証します。
- 解説文や総評文は `textContent` またはDOM APIで表示します。
- 外部入力由来の文字列を `innerHTML` に直接入れない方針です。
- 内部生成の固定ラベルやタグ表示では、一部 `innerHTML` を使っています。

## 制約と今後の改善候補

現在のAIはブラウザ内の同期処理で動いています。強いオセロ専用AIではなく、角・危険マス・合法手数・安定石などを使った手作り評価AIです。

今後の改善候補:

- AI探索のWeb Worker化
- 評価値グラフ
- スマホ向けの長文解説の折りたたみ/要点表示
- 良い手の時のネガティブタグ表示の調整
- `main.js` のモジュール分割
- ルール処理、AI、UI、JSON入出力の単体テスト
- より強いAI用のビットボード化や置換表
